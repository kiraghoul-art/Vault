How it works

The site runs on GitHub Pages (static). To talk to Discord, a small Cloudflare Worker acts as a secure proxy — your bot token lives there, never in the browser.

1
Go to discord.com/developers/applications → create an app → under Bot, enable it and copy the token.
2
In Discord → Settings → Advanced → enable Developer Mode. Then right-click each channel → Copy Channel ID.
3
Deploy the Cloudflare Worker from the /worker folder (see DEPLOY.md). Set DISCORD_TOKEN and ALLOWED_ORIGIN as environment variables.
4
Paste your Worker URL and channel IDs below. These are stored in localStorage — never sent anywhere except your own Worker.

Cloudflare Worker

The URL of your deployed Worker, e.g. https://kira-discord-proxy.yourname.workers.dev

Channel IDs

Right-click each channel in Discord (with Developer Mode on) → Copy ID.



Cloud Config — any device, one password

Save your entire configuration encrypted to a private Discord channel. On any new device, just enter the channel ID and your password to restore everything instantly.

1
Create a private channel in Discord, e.g. #vault-config — only you and the bot should have access.
2
Copy its ID (Developer Mode → right-click → Copy ID) and paste below.
3
Choose a strong password. Your config is AES-256 encrypted before being stored — even if someone reads the channel they can't decrypt it.
4
On a new device: enter the Worker URL, Config Channel ID, and password → click Load from Discord.
